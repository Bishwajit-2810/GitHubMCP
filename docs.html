Overview

GitHub MCP Server is a modular FastMCP server that exposes 12 tools over an SSE endpoint. Tools 1–10 wrap the GitHub REST and GraphQL APIs; Tools 11–12 power a RAG Q&A pipeline over the indexed codebase.

Quick Start

bash 
# 1. Clone and install
git clone <your-repo-url>
cd GitHubMCP

curl -LsSf https://astral.sh/uv/install.sh | sh   # install uv
uv venv && source .venv/bin/activate
uv pip install -e .

# 2. Configure
cp .env.example .env   # fill in GITHUB_TOKEN, GROQ_API_KEY, POSTGRES_URL …

# 3. Start PGVector (default vector store)
docker run -d --name pgvector \
  -e POSTGRES_USER=postgres -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=rag_db -p 5432:5432 pgvector/pgvector:pg16

# 4. Index the target repo into PGVector
python ingest.py

# 5. (Optional) verify RAG chain
python rag_query.py "Tell me about this project"

# 6. Start the MCP server
python server.py --port 8090

The server starts on http://localhost:8090/sse.

RAG Pipeline

The RAG pipeline fetches every file from the target GitHub repository (GITHUB_OWNER/GITHUB_REPO), embeds text chunks with all-MiniLM-L6-v2, and stores them in PGVector (default) or ChromaDB. The ask_codebase and explore_codebase tools query this store at runtime.

ℹ️
Vector store selection

Set RAG_VECTOR_DB in .env: pgvector (default) | chroma | both. Change it any time and restart the server — no re-indexing needed.

Step 1 — Install RAG dependencies

bash 
uv add langchain langchain-community langchain-chroma langchain-huggingface \
       langchain-groq langchain-text-splitters langchain-postgres \
       chromadb sentence-transformers pgvector psycopg2-binary

Step 2 — Index the repo (run once)

bash 
python ingest.py                 # full index → PGVector (default)
python ingest.py --reingest      # wipe & re-index
python ingest.py --db chroma     # ChromaDB only
python ingest.py --db both       # both stores
python ingest.py --docs-only     # local docs only (fast, no GitHub call)

What gets indexed

File Type Action
.py .md .js .ts .json .toml .yaml .html .css .sh .sql Split into 500-char chunks and embedded via sentence-transformers/all-MiniLM-L6-v2
.png .jpg .gif .svg .ico .webp .pdf .zip Stored as metadata stubs — not embedded, but visible to explore_codebase
⚠️
RAG_SKIP_LOCAL_DOCS

Set RAG_SKIP_LOCAL_DOCS=true in .env (recommended) to prevent the MCP project's own README.md, plan.md, and docs.html from being mixed in with the target repo's data.

Module

ingest.py

ingest.py

Fetches every file from the target GitHub repository (GITHUB_OWNER/GITHUB_REPO), embeds text chunks, and loads them into PGVector (default) and/or ChromaDB. Run once before using the RAG tools; re-run after repo changes with --reingest.

python — file docstring 
"""RAG Ingestion Script — indexes a GitHub repository into a vector store.

Loads TWO sources into the chosen vector store (PGVector by default):

  1. GitHub Repo  (controlled by GITHUB_OWNER / GITHUB_REPO in .env)
     - Text files (.py .md .js .ts .json .toml .yaml .html .css .sh .sql .txt)
       → split into 500-char chunks and embedded via HuggingFace MiniLM-L6-v2
     - Binary/image files (.png .jpg .gif .svg .webp .ico .bmp .pdf .zip)
       → stored as metadata stubs so explore_codebase can list them

  2. Local docs files  (skipped when RAG_SKIP_LOCAL_DOCS=true)
     - Any .html .md .txt file found in the project root

Usage:
    python ingest.py               # full index → PGVector (default)
    python ingest.py --reingest    # wipe store first, then re-index
    python ingest.py --db both     # PGVector + ChromaDB
    python ingest.py --docs-only   # local docs only (fast, no GitHub call)
"""

Function Reference

Function Returns Description
_gh_headers() dict GitHub REST API authentication headers
_get_default_branch() str Fetch default branch from GitHub API; falls back to "main"
_get_all_repo_files(branch) list[dict] Full recursive file tree via GitHub Git Trees API
_fetch_file_content(path, branch) str | None Download raw file content from GitHub raw URL
_strip_html(html) str Strip tags, decode HTML entities, return clean plain text
_ext(path) str Return lowercase extension with dot, e.g. .py, or ""
load_repo_documents(branch) (text_docs, binary_docs) Load all repo files; split into text documents + binary stubs
load_local_docs() list[Document] Auto-discover and load .html .md .txt .rst files from project tree
split_docs(docs) list[Document] 500-char chunks with 50-char overlap using RecursiveCharacterTextSplitter
get_embeddings() HuggingFaceEmbeddings Load sentence-transformers/all-MiniLM-L6-v2 embedding model
ingest_chroma(docs, emb) int Embed + persist to ChromaDB; returns stored doc count
ingest_pgvector(docs, emb) None Optionally embed + persist to PGVector (skipped if no POSTGRES_URL)
main() None CLI entry point: parse --reingest / --docs-only flags
bash — cli 
python ingest.py               # full index
python ingest.py --reingest    # wipe & re-index
python ingest.py --docs-only   # local docs only (fast)
