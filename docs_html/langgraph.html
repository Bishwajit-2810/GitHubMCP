🕸️ LangGraph & Agents

LangGraph extends LangChain with stateful, graph-based agent execution. Instead of linear chains, you model agent behaviour as a directed graph — with nodes that do work, edges that route between them, and a shared State that persists across steps.

✅ What you'll learn How to define tools with @tool, create agents with create_agent, understand the ReAct pattern, build a LangGraph state machine, add conditional routing, and compose multi-agent pipelines.

Overview

An agent is a system that uses an LLM to decide which actions to take. Unlike chains (which always follow the same steps), agents reason about what to do next based on the current state.

Why LangGraph?

Feature LangChain Chains LangGraph Agents
Execution flow Fixed, linear Dynamic, graph-based
Decision making Deterministic LLM-driven at each step
Loops / retries Not supported Native via cycle detection
Shared state Input/output only Typed state persists across all nodes
Multi-agent Manual composition First-class support

State

State is the shared data structure that every node in the graph reads from and writes to. It's typically a TypedDict where messages is a list that gets appended. 

from typing import TypedDict, Annotated
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages

# State definition — every node receives and returns this
class AgentState(TypedDict):
    # add_messages is a "reducer" — it appends new messages instead of overwriting
    messages: Annotated[list[BaseMessage], add_messages]
# State after user sends "What is 25+7?"
{
  "messages": [
    HumanMessage(content="What is 25+7?"), # user input
    AIMessage(content="", tool_calls=[{"name": "calculator", "args": {"query": "25+7"}}]),
    ToolMessage(content="32"), # tool result
    AIMessage(content="25 + 7 = 32") # final answer
  ]
}

Nodes

A node is a Python function that receives the current State and returns an update to apply to it. Nodes are where the actual work happens. 

from langgraph.graph import StateGraph, END
from langchain_groq import ChatGroq

llm = ChatGroq(model="openai/gpt-oss-120b")
tools = [calculator]  # list of @tool functions

# Bind tools to the LLM so it knows they exist
llm_with_tools = llm.bind_tools(tools)

# Node 1: The LLM decides what to do
def call_model(state: AgentState):
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}  # add the response to state

# Node 2: Execute whatever tool the LLM called
from langgraph.prebuilt import ToolNode
tool_node = ToolNode(tools)  # automatically handles tool dispatch

Edges

Edges define the flow between nodes. A normal edge always goes from A to B. A conditional edge calls a function to decide the next node. 

from langgraph.graph import StateGraph, END
from langgraph.prebuilt import tools_condition

# Build graph
graph = StateGraph(AgentState)
graph.add_node("agent", call_model)
graph.add_node("tools", tool_node)

graph.set_entry_point("agent")  # where execution starts

# Conditional: if LLM wants to call a tool → go to "tools", else → END
graph.add_conditional_edges(
    "agent",
    tools_condition,   # returns "tools" or "__end__"
)

# After tools run, always go back to the agent
graph.add_edge("tools", "agent")

app = graph.compile()

Defining Tools with @tool

The @tool decorator converts a regular Python function into a LangChain Tool. The docstring becomes the tool description that the LLM reads to decide when to use it. Type annotations define the schema. 

from langchain_core.tools import tool

@tool
def calculator(query: str) -> str:
    """Useful for solving math expressions. Input should be a valid Python math expression."""
    try:
        result = eval(query)   # safely evaluate math expressions
        return str(result)
    except Exception:
        return "Invalid expression."

# The tool has an auto-generated schema:
print(calculator.name)         # "calculator"
print(calculator.description)  # the docstring text
print(calculator.args)         # {"query": {"type": "string", "description": "..."}}
⚠️ Security note on eval() Using eval() on untrusted input is dangerous. In production, use a library like numexpr or simpleeval to safely evaluate math expressions without executing arbitrary code.

Tool Parameters

Attribute Set By Description
.name Function name The name LLM uses to call this tool
.description Docstring When and how to use this tool — LLM reads this
.args Type annotations JSON schema of expected arguments
.return_direct Decorator kwarg If True, return tool output directly without re-querying LLM

create_agent

create_agent is a convenience function from langchain.agents that builds a fully configured ReAct agent graph. Under the hood it creates a StateGraph with the correct nodes, edges, and tool routing. 

from langchain.agents import create_agent
from langchain_groq import ChatGroq
from langchain_core.tools import tool

@tool
def calculator(query: str) -> str:
    """Useful for solving math expressions."""
    try:
        return str(eval(query))
    except Exception:
        return "Invalid expression."

llm = ChatGroq(model="openai/gpt-oss-120b", temperature=0)

agent = create_agent(
    llm,                    # the language model
    tools=[calculator],     # list of available tools
    system_prompt="You are a helpful assistant with access to a calculator tool. Use it for math.",
)

response = agent.invoke({
    "messages": [{"role": "user", "content": "What is 25 + 7?"}]
})

# Extract the final AI message
final = response["messages"][-1]
print(final.content)  # "25 + 7 = 32"

create_agent Parameters

Parameter Type Description
llm BaseChatModel The LLM to use for reasoning decisions
tools list[BaseTool] All tools the agent can use
system_prompt str Initial system instruction for the agent
checkpointer BaseCheckpointSaver Optional: persist agent memory across runs

ReAct Pattern

ReAct (Reason + Act) is the dominant pattern for LLM agents. The LLM alternates between thinking about what to do and acting (calling a tool). The result of each action is fed back to the LLM for further reasoning.

Agent Memory

By default, agents are stateless — each .invoke() call is independent. Use a checkpointer to persist conversation history across calls. 

from langgraph.checkpoint.memory import MemorySaver
from langchain.agents import create_agent

checkpointer = MemorySaver()  # in-memory (for testing; use SqliteSaver for persistence)

agent = create_agent(llm, tools=tools, system_prompt="...", checkpointer=checkpointer)

# Thread ID groups a conversation
config = {"configurable": {"thread_id": "user-123"}}

# First turn
agent.invoke({"messages": [{"role": "user", "content": "Hi, I'm Alice."}]}, config=config)

# Second turn — the agent REMEMBERS Alice from the first turn
response = agent.invoke({"messages": [{"role": "user", "content": "What's my name?"}]}, config=config)
print(response["messages"][-1].content)  # "Your name is Alice!"

Conditional Edges

Conditional edges use a routing function to decide the next node. The function receives the current State and returns a node name (or END). 

from langgraph.graph import END

def should_continue(state: AgentState) -> str:
    """Route to tools if the last message has tool calls, otherwise end."""
    last = state["messages"][-1]
    if hasattr(last, "tool_calls") and last.tool_calls:
        return "tools"   # there's a tool call → execute it
    return END           # no tool call → we're done

graph.add_conditional_edges(
    "agent",          # source node
    should_continue,  # routing function
    {
        "tools": "tools",   # if function returns "tools", go to tools node
        END: END,           # if function returns END, finish
    }
)

Checkpointing

Saver Storage Best For
MemorySaver() In-process dict Testing and development
SqliteSaver.from_conn_string("...") SQLite file Single-machine persistence
PostgresSaver(...) PostgreSQL Production, multi-instance

Multi-Agent Graph

In a multi-agent setup, a supervisor agent routes tasks to specialised worker agents. Each worker is itself a LangGraph sub-graph.

📙 Task 1 — Basic Agent with Calculator

Goal: Create a basic agent using create_agent with a calculator tool. 
import os
from dotenv import load_dotenv
from langchain_groq import ChatGroq
from langchain_core.tools import tool
from langchain_core.messages import AIMessage
from langchain.agents import create_agent

load_dotenv()

@tool
def calculator(query: str) -> str:
    """Useful for solving math expressions."""
    try:
        return str(eval(query))
    except Exception:
        return "Invalid expression."

llm = ChatGroq(model="openai/gpt-oss-120b", temperature=0)

agent = create_agent(
    llm,
    tools=[calculator],
    system_prompt="You are a helpful assistant with access to a calculator tool. Use it for math questions.",
)

response = agent.invoke({
    "messages": [{"role": "user", "content": "What is 25 + 7?"}]
})

final_message = response["messages"][-1]
if isinstance(final_message, AIMessage):
    print(final_message.content)
else:
    print(final_message)

📙 Task 2 — Multi-Tool Agent

Goal: Build an agent with both weather and search tools and an interactive loop. 
import os, requests
from dotenv import load_dotenv
from langchain_groq import ChatGroq
from langchain_core.tools import tool
from langchain_core.messages import AIMessage
from langchain.agents import create_agent

load_dotenv()

@tool
def weather_api(city: str) -> str:
    """Get current weather for a given city."""
    api_key = os.getenv("OPENWEATHER_API_KEY")
    if not api_key:
        return "OpenWeather API key not found."
    url = "https://api.openweathermap.org/data/2.5/weather"
    params = {"q": city, "appid": api_key, "units": "metric"}
    resp = requests.get(url, params=params, timeout=10).json()
    if "main" not in resp:
        return f"Error: {resp.get('message', 'Unknown')}"
    return f"Weather in {city}: {resp['main']['temp']}°C, {resp['weather'][0]['description']}"

@tool
def google_search_api(query: str) -> str:
    """Search Google for current information. Returns top 3 results."""
    api_key = os.getenv("SERPER_API_KEY")
    if not api_key:
        return "Serper API key not found."
    headers = {"X-API-KEY": api_key, "Content-Type": "application/json"}
    data = requests.post("https://google.serper.dev/search", headers=headers, json={"q": query}, timeout=10).json()
    if "organic" not in data:
        return "No results found."
    return "\n".join([f"{r['title']} - {r['snippet']}" for r in data["organic"][:3]])

llm = ChatGroq(model="openai/gpt-oss-120b", temperature=0)
agent = create_agent(llm, tools=[weather_api, google_search_api],
    system_prompt="You are a research assistant. Use tools for real-time data.")

print("Agent ready! Type 'exit' to quit.")
while True:
    user_input = input("\nYou: ").strip()
    if user_input.lower() in ["exit", "quit"]:
        break
    response = agent.invoke({"messages": [{"role": "user", "content": user_input}]})
    messages = response["messages"]
    for msg in reversed(messages):
        if hasattr(msg, "content") and msg.content:
            print(f"\nAgent: {msg.content}")
            break

📙 Task 3 — Build a LangGraph from Scratch

Goal: Manually construct a StateGraph without using create_agent, to understand the internals. 
from typing import TypedDict, Annotated
from langchain_core.messages import BaseMessage
from langchain_core.tools import tool
from langchain_groq import ChatGroq
from langgraph.graph import StateGraph, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode, tools_condition
from dotenv import load_dotenv

load_dotenv()

# 1. Define state
class AgentState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]

# 2. Define tools
@tool
def calculator(query: str) -> str:
    """Solve math expressions."""
    try:
        return str(eval(query))
    except Exception:
        return "Invalid."

tools = [calculator]

# 3. Define model node
llm = ChatGroq(model="openai/gpt-oss-120b", temperature=0)
llm_with_tools = llm.bind_tools(tools)

def call_model(state: AgentState):
    return {"messages": [llm_with_tools.invoke(state["messages"])]}

# 4. Build graph
graph = StateGraph(AgentState)
graph.add_node("agent", call_model)
graph.add_node("tools", ToolNode(tools))
graph.set_entry_point("agent")
graph.add_conditional_edges("agent", tools_condition)
graph.add_edge("tools", "agent")
app = graph.compile()

# 5. Run it
result = app.invoke({"messages": [{"role": "user", "content": "What is 144 / 12?"}]})
print(result["messages"][-1].content)
