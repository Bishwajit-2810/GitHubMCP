🤖 Multi-Agent Systems

Multi-agent systems let you compose several specialised AI agents, each with their own tools and expertise, into a coordinated pipeline. A supervisor agent routes tasks to the right worker; workers execute and return results.

✅ What you'll learn How to define tools with @tool, create agents, add Loguru callback logging, build an interactive agent loop, and compose a supervisor + worker multi-agent system.

Overview

An agent system follows the ReAct loop: Reason → Act → Observe → Repeat. Multiple agents extend this by letting one agent call another as if it were a tool.

Agent Patterns

Pattern Structure Best For
Single Agent 1 LLM + N tools General assistants with multiple capabilities
Supervisor + Workers Supervisor routes to specialised agents Clear task separation, each agent optimised
Sequential Pipeline Agent A output → Agent B input Multi-step workflows (research → write → review)
Parallel Agents Multiple agents run simultaneously Independent sub-tasks that can be merged

When to Use Multi-Agent

🤔 Single Agent is enough when: the task can be completed with a handful of tools and the context window is large enough to hold all memory.
✅ Multi-Agent shines when:
  • Tasks have clearly distinct domains (e.g. research vs coding vs reviewing)
  • Parallel execution would save significant time
  • Each sub-task is complex enough to warrant its own specialisation
  • You need independent quality checks between steps

Defining Tools

Each agent gets a set of tools it can use. Tools are Python functions decorated with @tool. 

import os, requests
from langchain_core.tools import tool

@tool
def weather_api(city: str) -> str:
    """Get current weather information for a given city using OpenWeather API.
    
    Args:
        city: Name of the city (e.g. 'London', 'New York', 'Tokyo')
    """
    api_key = os.getenv("OPENWEATHER_API_KEY")
    if not api_key:
        return "OpenWeather API key not found in environment."
    
    url = "https://api.openweathermap.org/data/2.5/weather"
    params = {"q": city, "appid": api_key, "units": "metric"}
    
    try:
        response = requests.get(url, params=params, timeout=10)
        data = response.json()
        if response.status_code != 200:
            return f"Weather data unavailable: {data.get('message', 'Unknown error')}"
        temp = data["main"]["temp"]
        desc = data["weather"][0]["description"]
        return f"Weather in {city}: {temp}°C, {desc}"
    except Exception as e:
        return f"Weather API error: {str(e)}"


@tool
def google_search_api(query: str) -> str:
    """Search the web for current information using Serper API. Returns top 3 results.
    
    Args:
        query: Search query string
    """
    api_key = os.getenv("SERPER_API_KEY")
    if not api_key:
        return "Serper API key not found in environment."
    
    url = "https://google.serper.dev/search"
    headers = {"X-API-KEY": api_key, "Content-Type": "application/json"}
    
    try:
        response = requests.post(url, headers=headers, json={"q": query}, timeout=10)
        data = response.json()
        if "organic" not in data:
            return "No search results found."
        results = data["organic"][:3]
        return "\n".join([f"{r.get('title')} — {r.get('snippet')}" for r in results])
    except Exception as e:
        return f"Search API error: {str(e)}"

Extracting the Final Response

The agent returns a dict with a messages list. The last non-empty message is the final answer. 

from langchain_core.messages import AIMessage

def extract_final_answer(response: dict) -> str:
    """Extract the agent's final answer from the LangGraph state dict."""
    messages = response["messages"]
    # Walk backwards to find the last message with content
    for msg in reversed(messages):
        if hasattr(msg, "content") and msg.content:
            return msg.content
    return "No final answer found."

# Example usage
response = agent.invoke({
    "messages": [{"role": "user", "content": "What is the weather in Paris?"}]
})
answer = extract_final_answer(response)
print(answer)

Interactive Conversation Loop 

if __name__ == "__main__":
    print("Agent ready! Type 'exit' to quit.\n")

Loguru Logging Setup

Loguru is a modern logging library that's far simpler than Python's standard logging module. It writes to both console and files with automatic rotation. 

from loguru import logger

# Console output is on by default.
# Add a file sink with rotation and retention:
logger.add(
    "agent.log",          # log file path
    rotation="10 MB",     # create a new file when this one reaches 10MB
    retention="7 days",   # keep log files for 7 days
    level="INFO",         # minimum log level to write
    format="{time:YYYY-MM-DD HH:mm:ss} | {level} | {message}",
)

# Usage:
logger.debug("Debug message — not written if level=INFO")
logger.info("ℹ️  Info message")
logger.success("✅ Success message")
logger.warning("⚠️  Warning message")
logger.error("❌ Error message")
logger.critical("🚨 Critical message")

Loguru logger.add() Parameters

Parameter Type Description
sink str / Path / callable File path, or a callable like sys.stderr
rotation str / int When to start a new file: "10 MB", "1 day", "00:00"
retention str / int How long to keep old files: "7 days", 3 (number of files)
level str Minimum level: DEBUG, INFO, SUCCESS, WARNING, ERROR, CRITICAL
format str Log message format string with {time}, {level}, {message} etc.
compression str Compress old files: "gz", "zip"
backtrace bool Show full stack trace on exceptions (default: True)

AgentLogger — Callback Handler

Create a custom BaseCallbackHandler that logs every agent action, tool call, and completion to loguru. 

from langchain_core.callbacks import BaseCallbackHandler
from loguru import logger

class AgentLogger(BaseCallbackHandler):
    """Logs all agent lifecycle events via loguru."""

    def on_agent_action(self, action, **kwargs):
        """Called when the agent decides to call a tool."""
        logger.info(
            f"🛠️  Agent action: tool={action.tool!r} | input={action.tool_input!r}"
        )

    def on_agent_finish(self, finish, **kwargs):
        """Called when the agent produces its final answer."""
        output_len = len(str(finish)) if finish else 0
        logger.success(f"✅ Agent finished: {output_len} chars in response")

    def on_tool_start(self, serialized, input_str, **kwargs):
        """Called just before a tool is executed."""
        tool_name = serialized.get("name", "unknown")
        logger.info(f"🔧 Tool start: {tool_name!r} | input: {input_str!r}")

    def on_tool_end(self, output, **kwargs):
        """Called when a tool returns its result."""
        logger.success(f"✅ Tool end: {len(str(output))} chars output")

    def on_tool_error(self, error, **kwargs):
        """Called when a tool raises an exception."""
        logger.error(f"❌ Tool error: {error}")

Supervisor Pattern

A supervisor agent receives the user's request and decides which worker agent to call. In LangGraph, this is modelled as a node that conditionally routes to other nodes. 

from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated, Literal
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages
from langchain_groq import ChatGroq
from langchain.agents import create_agent

class MultiAgentState(TypedDict):
    messages: Annotated[list[BaseMessage], add_messages]
    next: str  # which agent to call next

llm = ChatGroq(model="openai/gpt-oss-120b", temperature=0)

# Create specialised worker agents
weather_agent = create_agent(llm, tools=[weather_api],
    system_prompt="You specialise in weather data. Use weather_api for all weather queries.")

search_agent = create_agent(llm, tools=[google_search_api],
    system_prompt="You specialise in web search. Use google_search_api for all lookup queries.")

# Supervisor uses routing prompt
from langchain_core.prompts import ChatPromptTemplate

supervisor_prompt = ChatPromptTemplate.from_messages([
    ("system", """You are a supervisor routing tasks to specialised agents.
Based on the user's question, respond with ONLY the agent name to call:
- 'weather_agent' — for weather, temperature, forecast questions
- 'search_agent' — for factual lookups, current events, general information
- 'FINISH' — if you can answer directly without any agent

Respond with ONLY the agent name or FINISH."""),
    ("human", "{input}"),
])

def supervisor_node(state: MultiAgentState) -> MultiAgentState:
    """Supervisor decides which agent to call."""
    last_message = state["messages"][-1]
    response = llm.invoke(supervisor_prompt.format_messages(input=last_message.content))
    return {"next": response.content.strip()}

Agent Handoffs

def weather_node(state: MultiAgentState) -> MultiAgentState:
    """Run the weather agent and return its response."""
    result = weather_agent.invoke({"messages": state["messages"]})
    return {"messages": result["messages"], "next": "supervisor"}

def search_node(state: MultiAgentState) -> MultiAgentState:
    """Run the search agent and return its response."""
    result = search_agent.invoke({"messages": state["messages"]})
    return {"messages": result["messages"], "next": "supervisor"}

def route(state: MultiAgentState) -> Literal["weather_agent", "search_agent", "__end__"]:
    """Route to the appropriate agent based on supervisor decision."""
    next_agent = state.get("next", "FINISH")
    if next_agent == "weather_agent":
        return "weather_agent"
    elif next_agent == "search_agent":
        return "search_agent"
    return "__end__"

Full Multi-Agent System

# Build the graph
graph = StateGraph(MultiAgentState)
graph.add_node("supervisor", supervisor_node)
graph.add_node("weather_agent", weather_node)
graph.add_node("search_agent", search_node)

graph.set_entry_point("supervisor")

graph.add_conditional_edges(
    "supervisor",
    route,
    {
        "weather_agent": "weather_agent",
        "search_agent": "search_agent",
        "__end__": END,
    },
)
# Workers always return to supervisor
graph.add_edge("weather_agent", "supervisor")
graph.add_edge("search_agent", "supervisor")

multi_agent_app = graph.compile()

# Run the full multi-agent system
result = multi_agent_app.invoke({
    "messages": [{"role": "user", "content": "What's the weather in Berlin?"}],
    "next": "",
})

for msg in result["messages"]:
    if hasattr(msg, "content") and msg.content:
        print(f"{msg.__class__.__name__}: {msg.content[:200]}")

# Save as simple_agent.py
import os, requests
from dotenv import load_dotenv
from langchain_groq import ChatGroq
from langchain_core.tools import tool
from langchain.agents import create_agent

load_dotenv()

@tool
def weather_api(city: str) -> str:
    """Get current weather for a city."""
    api_key = os.getenv("OPENWEATHER_API_KEY")
    if not api_key: return "No API key"
    resp = requests.get("https://api.openweathermap.org/data/2.5/weather",
        params={"q": city, "appid": api_key, "units": "metric"}, timeout=10).json()
    if "main" not in resp: return f"Error: {resp.get('message')}"
    return f"{city}: {resp['main']['temp']}°C, {resp['weather'][0]['description']}"

llm = ChatGroq(model="openai/gpt-oss-120b", temperature=0)
agent = create_agent(llm, tools=[weather_api], system_prompt="Use weather_api for weather questions.")

resp = agent.invoke({"messages": [{"role": "user", "content": "Weather in London?"}]})
print(resp["messages"][-1].content)

Ecosystem Integration: Tying it all together

Combine everything from this documentation into one system:

  1. LangChain — builds the prompt and chain for each agent
