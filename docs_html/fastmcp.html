⚡ FastMCP

FastMCP is a Pythonic framework that makes building MCP servers dramatically simpler. Where the raw mcp library requires boilerplate for every tool, FastMCP lets you decorate any Python function and have a fully compliant MCP server in minutes.

✅ FastMCP = Fast + MCP FastMCP handles all the protocol wiring (JSON-RPC, schema generation, transport setup) automatically. You focus on the logic; FastMCP handles the rest.

Overview

FastMCP vs Raw MCP

❌ Raw MCP SDK (verbose)

server = Server("my-server")

@server.list_tools()
async def list_tools():
    return [Tool(name="greet",
        description="Say hello",
        inputSchema={...})]

@server.call_tool()
async def call_tool(name, args):
    if name == "greet":
        return [TextContent(
            type="text",
            text=f"Hello, {args['name']}!"
        )]

async with stdio_server() as (r,w):
    await server.run(r, w, ...)

✅ FastMCP (clean)

from fastmcp import FastMCP

mcp = FastMCP("my-server")

@mcp.tool
def greet(name: str) -> str:
    """Say hello."""
    return f"Hello, {name}!"

mcp.run()

Installation

pip install fastmcp

FastMCP requires Python 3.10+. It can run both as a stdio server (for local tools) and as an HTTP server (for remote access).

Server Basics

Create a FastMCP instance to represent your server. This object has decorators for tools, resources, and prompts. 

from fastmcp import FastMCP

# Create the server — the name is shown to clients
mcp = FastMCP("My MCP Server")

# Configure server options inline
mcp = FastMCP(
    "My MCP Server",
    instructions="This server provides weather and search tools.",  # system instructions
    host="0.0.0.0",  # host for HTTP transport
    port=8080,       # port for HTTP transport
)

FastMCP Constructor Parameters

Parameter Type Default Description
name str required Server name shown to connecting clients
instructions str None Optional system-level instructions for the LLM
host str "127.0.0.1" Host for HTTP transport
port int 8000 Port for HTTP transport

@mcp.tool — Defining Tools

The @mcp.tool decorator registers any Python function as an MCP tool. Type annotations define the parameter schema; the docstring becomes the description. 

from fastmcp import FastMCP

mcp = FastMCP("My Server")

# Simple synchronous tool
@mcp.tool
def greet(name: str) -> str:
    """Say hello to someone by name.
    
    Args:
        name: The person's name to greet
    """
    return f"Hello, {name}!"

# Async tool — use async for I/O-bound operations
@mcp.tool
async def fetch_data(url: str, timeout: int = 10) -> str:
    """Fetch data from an HTTP URL.
    
    Args:
        url: The URL to fetch from
        timeout: Request timeout in seconds (default: 10)
    """
    import httpx
    async with httpx.AsyncClient() as client:
        response = await client.get(url, timeout=timeout)
        return response.text

# Tool with multiple parameters and optional fields
@mcp.tool
def search_users(
    query: str,
    limit: int = 10,
    active_only: bool = True
) -> str:
    """Search for users in the database.
    
    Args:
        query: Search term
        limit: Maximum number of results to return
        active_only: Whether to return only active users
    """
    return f"Found matching users for: {query} (limit={limit}, active={active_only})"

@mcp.tool Behaviour

Feature How FastMCP handles it
Tool name Taken from the Python function name
Description Taken from the Google-style docstring first line
Parameter schema Auto-generated from Python type annotations
Required params Parameters without default values are required
Optional params Parameters with defaults become optional in the schema
Return type str is wrapped in a TextContent response automatically

@mcp.resource — Exposing Data

Resources make data available to the LLM for context. They are identified by a URI and can be parameterised. 

import json

@mcp.resource("config://server")
def get_server_config() -> str:
    """Returns the current server configuration."""
    return json.dumps({
        "version": "1.0",
        "model": "openai/gpt-oss-120b",
        "features": ["tools", "resources", "prompts"]
    }, indent=2)

# Parameterised resource — URI template
@mcp.resource("user://{user_id}/profile")
async def get_user_profile(user_id: str) -> str:
    """Fetch a user's profile.
    
    Args:
        user_id: The unique user identifier
    """
    # In a real app, query a database here
    return json.dumps({"id": user_id, "name": "Alice", "role": "admin"})

@mcp.prompt — Reusable Prompts

Prompts are reusable message templates that clients can request. Useful for consistent system instructions or task-specific prompts. 

from mcp.types import TextContent

@mcp.prompt
def analyze_code_prompt(language: str, code: str) -> list[TextContent]:
    """Generate a prompt for code analysis.
    
    Args:
        language: Programming language (e.g. Python, JavaScript)
        code: The code to analyze
    """
    return [
        TextContent(type="text", text=f"""Analyze this {language} code:

```{language}
{code}
```

Provide:
1. What the code does
2. Potential bugs or issues
3. Suggestions for improvement
""")
    ]

The Context Object

FastMCP provides a Context object that your tool functions can accept. It gives you access to logging, progress reporting, and other server capabilities at runtime. 

from fastmcp import FastMCP, Context

mcp = FastMCP("My Server")

@mcp.tool
async def long_running_task(items: int, ctx: Context) -> str:
    """Process a bunch of items with progress reporting.
    
    Args:
        items: Number of items to process
    """
    results = []
    for i in range(items):
        # Report progress to the client
        await ctx.report_progress(i, items)
        
        # Log a message (visible to the client)
        await ctx.info(f"Processing item {i+1}/{items}")
        
        results.append(f"item_{i}")

    return f"Processed {len(results)} items"

Context Methods

Method Description
ctx.info(message) Send an info log message to the client
ctx.warning(message) Send a warning to the client
ctx.error(message) Send an error message to the client
ctx.report_progress(current, total) Report progress for long-running operations
ctx.read_resource(uri) Read a resource from the current server

Running the Server

Call mcp.run() with the desired transport. For local development use stdio; for network access use http. 

from fastmcp import FastMCP

mcp = FastMCP("My Server")

@mcp.tool
def greet(name: str) -> str:
    """Greet someone."""
    return f"Hello, {name}!"

# Run options:
if __name__ == "__main__":
    # Option 1: stdio (for local tools, default)
    mcp.run()                              # transport defaults to stdio

    # Option 2: HTTP server (network accessible)
    mcp.run(transport="http", port=8080)   # available at localhost:8080/mcp

    # Option 3: SSE (HTTP with streaming)
    mcp.run(transport="sse", port=8080)

mcp.run() Parameters

Parameter Type Default Description
transport str "stdio" "stdio", "http", or "sse"
host str "127.0.0.1" Bind address for HTTP/SSE transport
port int 8000 Port for HTTP/SSE transport
path str "/mcp" URL path for HTTP/SSE transport

FastMCP Client

FastMCP ships its own async client that connects to both HTTP and stdio servers with a clean context manager API. 

import asyncio
from fastmcp import Client

# Connect to an HTTP server
client = Client("http://localhost:8080/mcp")

async def main():
    async with client:
        # List all available tools
        tools = await client.list_tools()
        print("Tools:", [t.name for t in tools])

        # Call a tool by name with keyword arguments
        result = await client.call_tool("greet", {"name": "Alice"})
        print(result)  # prints the tool output

asyncio.run(main())

Client Methods

Method Returns Description
await client.list_tools() list[Tool] Get all tools the server exposes
await client.call_tool(name, args) list[Content] Call a tool with the given arguments
await client.list_resources() list[Resource] Get all resources the server exposes
await client.read_resource(uri) Content Read a specific resource by URI
await client.list_prompts() list[Prompt] Get all prompts the server exposes
await client.get_prompt(name, args) list[Message] Get a rendered prompt

Calling Tools from the Client 

import asyncio
from fastmcp import Client

async def call_tool(name: str):
    client = Client("http://localhost:8080/mcp")
    async with client:
        result = await client.call_tool("greet", {"name": name})
        # result is a list of content items
        for item in result:
            print(item.text)  # TextContent has a .text attribute

asyncio.run(call_tool("Bishwajit"))

Real-World Example: GitHub MCP Server

The github_mcp/server.py file in this project is a production FastMCP server that wraps the GitHub REST and GraphQL APIs. It demonstrates how to build a real-world MCP server with authentication, multiple tools, and error handling. 

import os
import httpx
from dotenv import load_dotenv
from loguru import logger
from fastmcp import FastMCP

load_dotenv()

mcp = FastMCP("github-mcp")

GITHUB_API = "https://api.github.com"
GITHUB_TOKEN = os.environ.get("GITHUB_TOKEN", "")
DEFAULT_OWNER = os.environ.get("GITHUB_OWNER", "")
DEFAULT_REPO = os.environ.get("GITHUB_REPO", "")

def _headers() -> dict:
    """Build authenticated GitHub API headers."""
    if not GITHUB_TOKEN:
        raise RuntimeError("GITHUB_TOKEN is not set. Add it to your .env file.")
    return {
        "Authorization": f"Bearer {GITHUB_TOKEN}",
        "Accept": "application/vnd.github+json",
        "X-GitHub-Api-Version": "2022-11-28",
    }

def _raise_for_status(resp: httpx.Response) -> None:
    """Raise a descriptive error for 4xx/5xx responses."""
    if resp.status_code >= 400:
        try:
            detail = resp.json()
        except Exception:
            detail = resp.text
        logger.error(f"GitHub API {resp.status_code}: {detail}")
        raise RuntimeError(f"GitHub API error {resp.status_code}: {detail}")

GitHub MCP Tools

Here's how the GitHub MCP server exposes repository and issue management as MCP tools: 

@mcp.tool
async def list_issues(
    owner: str = DEFAULT_OWNER,
    repo: str = DEFAULT_REPO,
    state: str = "open",
    limit: int = 20,
) -> str:
    """List GitHub issues for a repository.

    Args:
        owner: GitHub username or org (default from GITHUB_OWNER env)
        repo: Repository name (default from GITHUB_REPO env)
        state: Issue state — 'open', 'closed', or 'all'
        limit: Maximum number of issues to return
    """
    url = f"{GITHUB_API}/repos/{owner}/{repo}/issues"
    params = {"state": state, "per_page": limit}
    async with httpx.AsyncClient() as client:
        resp = await client.get(url, headers=_headers(), params=params)
        _raise_for_status(resp)
        issues = resp.json()
        if not issues:
            return f"No {state} issues found."
        lines = [f"#{i['number']}: {i['title']} [{i['state']}]" for i in issues]
        return "\n".join(lines)


@mcp.tool
async def create_issue(
    title: str,
    body: str = "",
    labels: list[str] | None = None,
    owner: str = DEFAULT_OWNER,
    repo: str = DEFAULT_REPO,
) -> str:
    """Create a new GitHub issue.

    Args:
        title: Issue title
        body: Issue body / description (markdown supported)
        labels: List of label names to add
        owner: GitHub username or org
        repo: Repository name
    """
    url = f"{GITHUB_API}/repos/{owner}/{repo}/issues"
    payload = {"title": title, "body": body}
    if labels:
        payload["labels"] = labels
    async with httpx.AsyncClient() as client:
        resp = await client.post(url, headers=_headers(), json=payload)
        _raise_for_status(resp)
        created = resp.json()
        return f"Created issue #{created['number']}: {created['html_url']}"


@mcp.tool
async def get_file_content(
    path: str,
    owner: str = DEFAULT_OWNER,
    repo: str = DEFAULT_REPO,
    ref: str = "main",
) -> str:
    """Get the content of a file from a GitHub repository.

    Args:
        path: File path in the repo (e.g. 'src/main.py')
        owner: GitHub username or org
        repo: Repository name
        ref: Branch, tag, or commit SHA (default: 'main')
    """
    import base64
    url = f"{GITHUB_API}/repos/{owner}/{repo}/contents/{path}"
    params = {"ref": ref}
    async with httpx.AsyncClient() as client:
        resp = await client.get(url, headers=_headers(), params=params)
        _raise_for_status(resp)
        data = resp.json()
        if data.get("encoding") == "base64":
            return base64.b64decode(data["content"]).decode("utf-8")
        return data.get("content", "")


if __name__ == "__main__":
    mcp.run()  # stdio by default — connect with any MCP client

📙 Task 2 — HTTP Server with Multiple Tools

Goal: Add calculator and info tools, run over HTTP, and call each from the client. 
# server.py
from fastmcp import FastMCP
import platform, sys

mcp = FastMCP("MultiTool Server")

@mcp.tool
def calculate(expression: str) -> str:
    """Evaluate a math expression safely."""
    allowed = set("0123456789+-*/()., ")
    if not all(c in allowed for c in expression):
        return "Error: invalid characters in expression"
    try:
        return str(eval(expression))
    except Exception as e:
        return f"Error: {e}"

@mcp.tool
def server_info() -> str:
    """Get information about the server environment."""
    return (f"Python {sys.version}\n"
            f"Platform: {platform.system()} {platform.release()}")

@mcp.resource("docs://readme")
def get_readme() -> str:
    """Returns the server documentation."""
    return "# MultiTool Server\nProvides calculator and server-info tools."

if __name__ == "__main__":
    mcp.run(transport="http", port=8080)
