🔗 LangChain

LangChain is the foundational framework for building LLM-powered applications. It provides composable building blocks — prompts, models, parsers, and chains — that you connect together using the LCEL pipe operator.

✅ What you'll learn How to create prompt templates, connect them to ChatGroq, parse outputs, compose multi-step chains, and instrument them with callbacks — from zero to production.

Overview

LangChain follows a simple mental model: everything is a Runnable. A Runnable accepts an input and produces an output. You compose Runnables using the pipe operator | to form chains.

Key Components

Component Class Role
LLM / Chat Model ChatGroq Sends prompts to Groq and returns AI responses
Prompt Template ChatPromptTemplate, PromptTemplate Formats dynamic input variables into a prompt string
Output Parser StrOutputParser Converts the raw AI response object to a plain string
Chain Runnable (via |) Composes steps into a pipeline
Callback Handler BaseCallbackHandler Hooks into chain execution for logging / monitoring

Installation

pip install langchain langchain-groq langchain-core langchain-community python-dotenv

Create a .env file with your Groq API key:

GROQ_API_KEY=your_api_key_here

ChatGroq — Model Setup

ChatGroq is the LangChain integration for Groq's ultra-fast inference API. It behaves exactly like any other LangChain chat model. 

from langchain_groq import ChatGroq
from langchain_core.messages import HumanMessage
from dotenv import load_dotenv

load_dotenv()  # loads GROQ_API_KEY from .env

llm = ChatGroq(
    model="openai/gpt-oss-120b",  # model identifier on Groq
    temperature=0.7,              # 0 = deterministic, 1 = creative
)

# Direct invocation with a message
response = llm.invoke([HumanMessage(content="Explain AI in one sentence.")])
print(response.content)

ChatGroq Parameters

Parameter Type Default Description
model str required Groq model identifier, e.g. "openai/gpt-oss-120b"
temperature float 0.7 Randomness: 0 = deterministic, 1 = very creative. Lower for factual tasks.
max_tokens int None Maximum tokens in the response. Useful for limiting costs.
groq_api_key str from env Explicit API key. Prefer setting via GROQ_API_KEY env var.
streaming bool False Stream tokens as they are generated. Use .stream() for this.

LCEL — LangChain Expression Language

LCEL is the pipe-based composition system. Each step in the pipe must be a Runnable. The output of one step becomes the input of the next. 

from langchain_core.prompts import ChatPromptTemplate
from langchain_groq import ChatGroq
from langchain_core.output_parsers import StrOutputParser

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are an expert AI tutor."),
    ("human", "Explain {topic} in simple terms."),
])

llm = ChatGroq(model="openai/gpt-oss-120b")
parser = StrOutputParser()

# Build chain with the pipe operator
chain = prompt | llm | parser

# Invoke it
result = chain.invoke({"topic": "machine learning"})
print(result)  # plain string output
💡 How the pipe operator works chain = prompt | llm | parser creates a RunnableSequence. When you call chain.invoke({"topic": "..."}, it:
  1. Formats the prompt with your variables
  2. Sends the formatted prompt to ChatGroq
  3. Extracts the text content from the response

Prompt Templates

Prompt templates separate your instructions from runtime data. They take a dict of variables and produce a formatted prompt.

ChatPromptTemplate

For chat models (recommended). Takes a list of (role, template) tuples. 

from langchain_core.prompts import ChatPromptTemplate

# System + Human message structure
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a {role}. Always be concise."),
    ("human", "{question}"),
])

# Gets formatted into a list of chat messages
formatted = prompt.format_messages(role="historian", question="Who was Napoleon?")
print(formatted)
# [SystemMessage(content='You are a historian...'), HumanMessage(content='Who was Napoleon?')]

PromptTemplate (simple string)

For simpler, non-chat templates. Used when chaining the output of one LLM into another. 

from langchain_core.prompts import PromptTemplate

# Uses {variable} placeholders
prompt = PromptTemplate.from_template(
    "Simplify this explanation further for beginners:\n\n{simple_explanation}"
)

# Preview the prompt
formatted = prompt.format(simple_explanation="AI learns from data using math...")
print(formatted)

Prompt Template Parameters

Method Use Case
from_messages([...]) Chat models — list of (role, template) tuples
from_template("...") Simple text template with {variable} placeholders
.format_messages(**vars) Preview the formatted messages
.input_variables List of variable names expected by the template

Output Parsers

The LLM returns an AIMessage object. Parsers convert it into a more useful format.

Parser Output Type Best For
StrOutputParser() str Plain text — most common
JsonOutputParser() dict Structured JSON responses
PydanticOutputParser() Pydantic model Typed, validated responses
CommaSeparatedListOutputParser() list[str] Bullet-list style outputs 
from langchain_core.output_parsers import StrOutputParser

parser = StrOutputParser()

# Inside a chain, it converts AIMessage → str automatically
chain = prompt | llm | parser

# You can also call it manually:
from langchain_core.messages import AIMessage
msg = AIMessage(content="Hello from the AI!")
print(parser.invoke(msg))  # "Hello from the AI!"

Building Chains

A chain is any combination of Runnables connected with |. The simplest chain is prompt | llm. Add a parser for clean output. 

from langchain_core.prompts import ChatPromptTemplate
from langchain_groq import ChatGroq
from langchain_core.output_parsers import StrOutputParser
from dotenv import load_dotenv

load_dotenv()

# Define each component
prompt = ChatPromptTemplate.from_messages([
    ("system", "You are an expert AI tutor."),
    ("human", "Explain {topic} in simple terms."),
])
llm = ChatGroq(model="openai/gpt-oss-120b")
parser = StrOutputParser()

# Compose the chain
chain = prompt | llm | parser

# Run it
response = chain.invoke({"topic": "neural networks"})
print(response)  # ✅ clean string, no AIMessage wrapper

Multi-Chain Composition

You can chain the output of one LLM call directly into the next. This is useful for progressive refinement, translation pipelines, or multi-step reasoning. 

from langchain_core.prompts import ChatPromptTemplate, PromptTemplate
from langchain_groq import ChatGroq
from langchain_core.output_parsers import StrOutputParser

# Stage 1: Expert explains a topic
prompt1 = ChatPromptTemplate.from_messages([
    ("system", "You are an expert AI tutor."),
    ("human", "Explain {topic} in simple terms."),
])
llm = ChatGroq(model="openai/gpt-oss-120b")
chain1 = prompt1 | llm  # output: AIMessage

# Stage 2: Simplify even further
# Note: the output of chain1 (AIMessage) is passed as the input — use {simple_explanation}
prompt2 = PromptTemplate.from_template(
    "Simplify this explanation further for beginners:\n\n{simple_explanation}"
)
parser = StrOutputParser()

# Full pipeline: chain1 output feeds into prompt2
chain2 = chain1 | prompt2 | llm | parser

response = chain2.invoke({"topic": "machine learning"})
print(response)
⚠️ Variable naming in chained prompts When the output of one chain feeds into a PromptTemplate, the template variable name must match what the previous step produces. Since ChatGroq returns an AIMessage, wrap it with StrOutputParser() first and rename the variable to match, or use RunnablePassthrough to restructure inputs.

Callback Handlers

Callbacks let you hook into chain lifecycle events for logging, monitoring, and debugging. Implement BaseCallbackHandler and override the methods you need.

Available Callback Methods

Method Triggered When
on_chain_start(serialized, inputs) A chain begins execution
on_chain_end(outputs) A chain completes successfully
on_llm_start(serialized, prompts) An LLM call is about to be made
on_llm_end(response) An LLM call completes
on_llm_error(error) An LLM call fails
on_tool_start(serialized, input_str) A tool is invoked by an agent
on_tool_end(output) A tool returns a result 
from langchain_core.callbacks import BaseCallbackHandler
from loguru import logger

class LoguruHandler(BaseCallbackHandler):
    """Custom callback handler that logs chain events via loguru."""

    def on_chain_start(self, serialized, inputs, **kwargs):
        logger.info(f"🚀 Chain started: {serialized['name']}, input: {inputs}")

    def on_llm_start(self, serialized, prompts, **kwargs):
        # Show only first 100 chars of the prompt to avoid log spam
        logger.info(f"🤖 LLM started: {serialized['name']}, prompt: {prompts[0][:100]}...")

    def on_llm_end(self, output, **kwargs):
        logger.success(f"✅ LLM completed: {len(str(output))} chars")

    def on_chain_end(self, output, **kwargs):
        logger.success(f"✅ Chain completed: {len(str(output))} chars")

RunnableConfig

RunnableConfig is passed as a second argument to .invoke(). It carries callbacks, tags, and metadata for a single execution. 

from langchain_core.runnables import RunnableConfig
import time

handler = LoguruHandler()

config = RunnableConfig(
    callbacks=[handler],       # list of callback handlers
    tags=["tutorial", "demo"], # labels for filtering in logs
    metadata={"user": "alice"} # arbitrary key-value pairs
)

start = time.time()
response = chain.invoke({"topic": "deep learning"}, config=config)
logger.info(f"⏱️ Done in {time.time() - start:.2f}s")

RunnableConfig Parameters

Parameter Type Description
callbacks list Callback handler instances to attach for this run
tags list[str] String labels for grouping / filtering traces
metadata dict Arbitrary key-value pairs passed to every callback
run_name str Human-readable name for this run in traces
max_concurrency int Max parallel calls in a batch() invocation

📘 Task 2 — Multi-Chain with Loguru Logging

Goal: Chain two LLM calls and instrument with a callback handler and loguru file logging. 
from loguru import logger
import time
from langchain_core.callbacks import BaseCallbackHandler
from langchain_core.runnables import RunnableConfig
from langchain_core.prompts import ChatPromptTemplate, PromptTemplate
from langchain_groq import ChatGroq
from langchain_core.output_parsers import StrOutputParser
from dotenv import load_dotenv

load_dotenv()

# -- File logging setup --
logger.add("chain.log", rotation="500 MB", retention="10 days", level="INFO")

class LoguruHandler(BaseCallbackHandler):
    def on_chain_start(self, serialized, inputs, **kwargs):
        logger.info(f"🚀 Chain: {serialized['name']} | {inputs}")
    def on_llm_end(self, output, **kwargs):
        logger.success(f"✅ LLM done: {len(str(output))} chars")

# -- Stage 1: expert explanation --
prompt1 = ChatPromptTemplate.from_messages([
    ("system", "You are an expert AI tutor."),
    ("human", "Explain {topic} in simple terms."),
])
llm = ChatGroq(model="openai/gpt-oss-120b")
chain1 = prompt1 | llm

# -- Stage 2: beginner simplification --
prompt2 = PromptTemplate.from_template(
    "Simplify this explanation further for beginners:\n\n{simple_explanation}"
)
chain2 = chain1 | prompt2 | llm | StrOutputParser()

config = RunnableConfig(callbacks=[LoguruHandler()], tags=["tutorial"])
start = time.time()
response = chain2.invoke({"topic": "machine learning"}, config=config)
logger.info(f"⏱️  Done in {time.time()-start:.2f}s")
print("Final:", response)

📘 Task 3 — Streaming Responses

Goal: Stream tokens as they are generated instead of waiting for the full response. 
from langchain_core.prompts import ChatPromptTemplate
from langchain_groq import ChatGroq
from langchain_core.output_parsers import StrOutputParser
from dotenv import load_dotenv

load_dotenv()

prompt = ChatPromptTemplate.from_messages([
    ("system", "You are a storyteller."),
    ("human", "Tell me a short story about {subject}."),
])
llm = ChatGroq(model="openai/gpt-oss-120b", streaming=True)
chain = prompt | llm | StrOutputParser()

# .stream() yields tokens as they arrive
for chunk in chain.stream({"subject": "a robot learning to paint"}):
    print(chunk, end="", flush=True)
print()  # newline at end
