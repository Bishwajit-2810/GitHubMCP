🔌 Model Context Protocol (MCP)

MCP is an open standard by Anthropic that defines how AI models communicate with external tools and data sources. Think of it as a "USB-C for AI" — one universal protocol that any model can use to talk to any tool.

🔑 Key Insight MCP separates the AI model from its tools. Instead of hardcoding tool integrations inside each model, MCP lets you build standalone tool servers that any MCP-compatible client can connect to.

Overview

The MCP Protocol

MCP uses JSON-RPC 2.0 as its message format. All communication follows a request/response pattern with optional notifications.

Message Type Direction Example
Request Client → Server {"method": "tools/call", "params": {...}}
Response Server → Client {"result": {"content": [...]}}
Notification Either direction {"method": "notifications/progress", "params": {...}}

MCP Primitives

MCP servers expose three types of primitives that clients can discover and use:

Primitive Purpose Example
Tools Functions the LLM can call to take actions get_weather(city), search_web(query)
Resources Data sources the LLM can read (files, APIs) file://config.json, db://users
Prompts Reusable prompt templates the LLM can request summarize_code, explain_error

Transport Types

📡 stdio

Server runs as a child process. Client communicates via stdin/stdout pipes. Great for local tools and development. 

StdioServerParameters(
    command="python",
    args=["mcp_servers.py"],
    env=None,
)

🌐 HTTP / SSE

Server runs as a standalone HTTP server. Client connects over the network. Good for remote tools and production deployments. 

mcp.run(
    transport="http",
    port=8080
)
Transport Process Model Best For
stdio Child process, pipes Local tools, scripts, development
http Standalone HTTP server Production, remote access, multiple clients
sse HTTP + Server-Sent Events Streaming results over HTTP

Server Setup

An MCP server registers its tools and then runs. Using the lower-level mcp library: 

from typing import Any
import httpx
from mcp.server.fastmcp import FastMCP  # FastMCP is part of the mcp package too
from dotenv import load_dotenv
from loguru import logger

load_dotenv()

# Create the server with a name
mcp = FastMCP("weather")

NWS_API_BASE = "https://api.weather.gov"
USER_AGENT = "weather-app/1.0"

async def make_nws_request(url: str) -> dict[str, Any] | None:
    """Make a request to the NWS API."""
    headers = {"User-Agent": USER_AGENT, "Accept": "application/geo+json"}
    async with httpx.AsyncClient() as client:
        try:
            response = await client.get(url, headers=headers, timeout=30.0)
            response.raise_for_status()
            return response.json()
        except Exception as e:
            logger.error("NWS request failed for {}: {}", url, e)
            return None

Defining Tools on a Server

Decorate async functions with @mcp.tool(). The docstring and type annotations generate the JSON schema that clients see. 

@mcp.tool()
async def get_alerts(state: str) -> str:
    """Get weather alerts for a US state.

    Args:
        state: Two-letter US state code (e.g. CA, NY)
    """
    url = f"{NWS_API_BASE}/alerts/active/area/{state}"
    data = await make_nws_request(url)

    if not data or "features" not in data:
        return "Unable to fetch alerts or no alerts found."
    if not data["features"]:
        return "No active alerts for this state."

    alerts = []
    for feature in data["features"]:
        props = feature["properties"]
        alerts.append(
            f"Event: {props.get('event', '?')}\n"
            f"Area: {props.get('areaDesc', '?')}\n"
            f"Severity: {props.get('severity', '?')}"
        )
    return "\n---\n".join(alerts)


@mcp.tool()
async def get_forecast(latitude: float, longitude: float) -> str:
    """Get weather forecast for a location.

    Args:
        latitude: Latitude of the location
        longitude: Longitude of the location
    """
    points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
    points_data = await make_nws_request(points_url)
    if not points_data:
        return "Unable to fetch forecast data."

    forecast_url = points_data["properties"]["forecast"]
    forecast_data = await make_nws_request(forecast_url)
    if not forecast_data:
        return "Unable to fetch detailed forecast."

    periods = forecast_data["properties"]["periods"]
    results = []
    for period in periods[:5]:
        results.append(
            f"{period['name']}: {period['temperature']}°{period['temperatureUnit']}, "
            f"{period['detailedForecast']}"
        )
    return "\n".join(results)


def main():
    mcp.run(transport="stdio")  # start the server

if __name__ == "__main__":
    main()

@mcp.tool() Parameters

Parameter Type Description
(none required) — The decorator reads name/description/args from the function automatically
name str Override the tool name (default: function name)
description str Override the tool description (default: docstring)

Resources

Resources are data sources exposed by the server. Unlike tools (which perform actions), resources are read-only data the LLM can request context from. 

from mcp.server.fastmcp import FastMCP, Context
import json

mcp = FastMCP("my-server")

@mcp.resource("config://app")
async def get_app_config() -> str:
    """Returns the current application configuration."""
    config = {"version": "1.0.0", "debug": False, "model": "gpt-oss-120b"}
    return json.dumps(config, indent=2)

@mcp.resource("file://{path}")
async def read_file(path: str) -> str:
    """Read a file from the server filesystem."""
    with open(path) as f:
        return f.read()

Server Registry

The server registry pattern lets you define multiple MCP servers in one place and reference them by name from any client. 

from mcp import StdioServerParameters
import os

# mcp_servers.py — the registry
SERVERS: dict[str, StdioServerParameters] = {
    "weather": StdioServerParameters(
        command="python",
        args=["mcp_servers.py"],  # path to the server script
        env=None,                 # environment variables (None = inherit)
    ),
    "magic": StdioServerParameters(
        command="npx",
        args=["-y", "@21st-dev/magic@latest"],
        env={**os.environ, "API_KEY": os.environ.get("MAGIC_API_KEY", "")},
    ),
}

StdioServerParameters

Parameter Type Description
command str Executable to run ("python", "npx", etc.)
args list[str] Arguments to pass to the command
env dict | None Environment variables. None = inherit from parent process
cwd str | None Working directory for the server process

Client Setup

The MCP client connects to a server, initialises the session, and can then list and call tools. 

import asyncio
from contextlib import AsyncExitStack
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

class MCPClient:
    def __init__(self):
        self.session = None
        self.exit_stack = AsyncExitStack()

    async def connect(self, server_params: StdioServerParameters):
        # Start the server process and get a stdio transport
        stdio_transport = await self.exit_stack.enter_async_context(
            stdio_client(server_params)
        )
        stdio, write = stdio_transport

        # Create and initialise the session
        self.session = await self.exit_stack.enter_async_context(
            ClientSession(stdio, write)
        )
        await self.session.initialize()
        print("Connected!")

Listing Tools

    async def list_available_tools(self):
        response = await self.session.list_tools()
        for tool in response.tools:
            print(f"Tool: {tool.name}")
            print(f"  Description: {tool.description}")
            print(f"  Schema: {tool.inputSchema}")
Tool Field Type Description
tool.name str The tool's identifier used to call it
tool.description str What the tool does (from docstring)
tool.inputSchema dict JSON Schema describing expected arguments

Calling Tools

    async def call_tool(self, tool_name: str, arguments: dict):
        result = await self.session.call_tool(tool_name, arguments)
        # result.content is a list of TextContent / ImageContent
        for item in result.content:
            print(item.text)
        return result

Full Groq + MCP Integration

The complete pattern: convert MCP tools to OpenAI function format, pass them to Groq, process tool calls in a loop, and return the final answer. 

from groq import Groq
import json

async def process_query(self, query: str) -> str:
    """Send a query to Groq, using MCP tools for execution."""
    messages = [{"role": "user", "content": query}]

    # 1. Get tools from MCP server
    tool_response = await self.session.list_tools()
    available_tools = [
        {
            "type": "function",
            "function": {
                "name": t.name,
                "description": t.description,
                "parameters": dict(t.inputSchema),  # JSON Schema
            },
        }
        for t in tool_response.tools
    ]

    # 2. Send to Groq with tools available
    response = self.groq.chat.completions.create(
        model="openai/gpt-oss-120b",
        max_tokens=1000,
        messages=messages,
        tools=available_tools,
        tool_choice="auto",         # let LLM decide whether to use tools
        parallel_tool_calls=False,  # call one tool at a time
    )

    # 3. Loop: process tool calls until LLM is done
    final_text = []
    while True:
        choice = response.choices[0]
        message = choice.message

        if message.content:
            final_text.append(message.content)

        if not message.tool_calls:
            break  # no more tool calls → done

        # 4. Execute each tool call via MCP
        messages.append({
            "role": "assistant",
            "content": message.content or "",
            "tool_calls": [
                {"id": tc.id, "type": "function",
                 "function": {"name": tc.function.name, "arguments": tc.function.arguments}}
                for tc in message.tool_calls
            ],
        })

        for tc in message.tool_calls:
            args = json.loads(tc.function.arguments)
            result = await self.session.call_tool(tc.function.name, args)
            tool_text = "\n".join(c.text for c in result.content if hasattr(c, "text"))

            messages.append({
                "role": "tool",
                "tool_call_id": tc.id,
                "content": tool_text,
            })

        # 5. Send updated history back to Groq
        response = self.groq.chat.completions.create(
            model="openai/gpt-oss-120b",
            max_tokens=1000,
            messages=messages,
            tools=available_tools,
            tool_choice="auto",
        )

    return "\n".join(final_text)

📗 Task 2 — Connect an MCP Client

Goal: Write a client that connects to the weather server, lists tools, and calls get_alerts. 
import asyncio
from contextlib import AsyncExitStack
from mcp import ClientSession, StdioServerParameters
from mcp.client.stdio import stdio_client

server_params = StdioServerParameters(command="python", args=["mcp_servers.py"])

async def main():
    async with AsyncExitStack() as stack:
        # Connect
        stdio, write = await stack.enter_async_context(stdio_client(server_params))
        session = await stack.enter_async_context(ClientSession(stdio, write))
        await session.initialize()

        # List tools
        tools = (await session.list_tools()).tools
        print("Available tools:", [t.name for t in tools])

        # Call get_alerts for California
        result = await session.call_tool("get_alerts", {"state": "CA"})
        for item in result.content:
            print(item.text)

asyncio.run(main())
